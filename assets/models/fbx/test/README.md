# FBX Unit Test Assets

This directory contains minimal FBX files generated by Blender scripts in
`tools/blender/`. Each file is designed to isolate and verify a single
aspect of FBX loading and animation playback.

---

## anim_euler_jump.fbx

**Test: Euler ±180° crossing (Euler jump)**

### Setup

- 1 armature, 1 bone (`Bone`), horizontal bar mesh skinned to it
- 3 sparse keyframes at frames 1 / 11 / 21 with **linear** interpolation:

  | Frame | Y rotation |
  |------:|----------:|
  |     1 |        0° |
  |    11 |      175° |
  |    21 |     −175° |

### What it tests

Between frames 11 and 21 the actual rotation change is only **10°** (the
short arc from 175° to −175°). However, if a loader interpolates the Euler
angles linearly it will take the long way around and produce a **350° sweep
in the wrong direction**.

| Behaviour | Frames 11→21 |
|-----------|-------------|
| Correct (quaternion slerp) | bar barely moves (~10° arc) |
| Incorrect (linear Euler) | bar sweeps 350° in the wrong direction |

### Generation

```
blender --background --python tools/blender/gen_anim_euler_jump.py
```

> Note: Blender's FBX bake automatically wraps −175° to 185° to minimise
> angular differences. The generation script applies a binary patch after
> export to restore the intended −175° keyframe value.

## anim_skin_bend.fbx

**Test: Basic 2-bone skinned mesh animation**

### Setup

- 1 armature with 2 connected bones:
  - `LowerBone`: 0–0.5 m (stationary)
  - `UpperBone`: 0.5–1.0 m (animated)
- 1 m cylinder mesh, vertices split by weight:
  - lower half → `LowerBone` (weight 1.0)
  - upper half → `UpperBone` (weight 1.0)
- 60 frames at 30 fps; `UpperBone` X-rotation: **0° → 90° → 0°**

### What it tests

Verifies that skinning (Deformer / Cluster) is correctly parsed and applied
so the mesh deforms smoothly with the bone hierarchy.

| Behaviour | Result |
|-----------|--------|
| Correct | cylinder bends like an elbow — smooth deformation |
| Incorrect | vertices fly away, or the upper half stays frozen |

### Generation

```
blender --background --python tools/blender/gen_anim_skin_bend.py
```

## anim_root_motion.fbx

**Test: Root bone translation and rotation**

### Setup

- 1 armature with 2 connected bones:
  - `RootBone`: 0–0.4 m — translates and rotates
  - `ChildBone`: 0.4–0.8 m — fixed 45° X-bend throughout
- Elongated box mesh fully weighted to `RootBone`
- 60 frames at 30 fps:
  - `RootBone` X-translation: **0 → 1 m** (100 FBX units)
  - `RootBone` Z-rotation: **0° → 360°** (full spin)
  - `ChildBone`: constant 45° X-rotation

### What it tests

Verifies that root bone **translation** is correctly extracted from FBX and
applied at runtime, combined with rotation. A child bone's local transform
should compose on top of the animated root.

| Behaviour | Result |
|-----------|--------|
| Correct | box moves to the right while spinning; child bone visible as an overhang |
| Incorrect | box stays at origin, moves in the wrong direction, or child bone detaches |

### Generation

```
blender --background --python tools/blender/gen_anim_root_motion.py
```

## Running all generators

```powershell
cd tools\blender
.\run_all.ps1
```

Blender 5.1 must be installed at the default path, or pass the executable
path as an argument: `.\run_all.ps1 'C:\...\blender.exe'`
